# 모바일 HUD 레이아웃 원칙

## 목표

모바일에서 `게임 화면이 먼저 보이고`, UI는 `작고 빠르게` 접근하게 만든다.
상세 패널은 숨기고, 전투 중 필요한 정보만 상시 노출한다.

## 상단 레이아웃

### 좌측 블록

- 캐릭터 초상
- 레벨
- HP 바
- MP 바
- 전투력

원칙:

- 세로 2줄 구성
- 초상과 수치 덩어리를 최대한 붙여서 폭을 줄임
- 상태창 높이는 캐릭터 2마리 정도 크기를 넘기지 않음

### 중앙 블록

- 미니 퀘스트 바
- 진행중 퀘스트 1줄
- `AUTO` 작은 토글

원칙:

- 상세 퀘스트 창은 숨김
- 현재 목적지만 1줄 노출
- 자동 길찾기 상태를 텍스트로 짧게 표시

### 우측 블록

- 골드
- 보석
- 우편
- 설정
- 던전
- 상점
- 메뉴

원칙:

- 개별 큰 버튼을 늘어놓지 말고 작은 원형/사각형 버튼 사용
- 자주 안 쓰는 메뉴는 접는 구조
- 추후 PvP, 이벤트, 캐시 상점을 같은 메뉴 묶음에 편입

## 하단 레이아웃

### 좌측

- 조이스틱

### 중앙

- 스킬 4~5칸
- 쿨타임 숫자
- 스킬북 드랍 아이콘과 같은 계열 프레임 사용

### 우측

- 기본 공격
- 회피 또는 특수 버튼
- Auto 버튼

### 최하단

- 얇은 경험치 바

## 패널 진입 방식

- 인벤토리
- 장비
- 스탯
- 스킬북
- 우편함
- 던전
- 설정
- 포인트 / 캐시

이 항목은 전부 `버튼 진입형`으로 처리한다.
상시 박스를 띄우지 않는다.

## PWA 기준

- 상단은 `safe-area-inset-top` 고려
- 하단 스킬바는 `safe-area-inset-bottom` 고려
- 좌우 모서리 버튼은 베젤/노치 여백 확보
- 세로 모드는 안내 오버레이만 띄우고 실제 플레이는 가로 고정
