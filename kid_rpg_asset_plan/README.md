# kid_rpg_asset_plan

`kid_rpg` 본작업 전에 먼저 검수할 에셋 계획 폴더입니다.

목표 방향:

- 모바일 ARPG 느낌의 도트 2.5D 화면
- 어두운 던전/필드 분위기
- 실제 게임 같은 HUD와 스킬바
- 4방향 캐릭터, 몬스터, NPC, 타일맵, 구조물, 스킬 이펙트, 아이템 아이콘을 먼저 고정

구성:

- [ASSET_SELECTION.md](./ASSET_SELECTION.md): 카테고리별 우선 후보와 사용 이유
- [sources.json](./sources.json): 소스 정리용 manifest
- `characters_npc/`: 플레이어, 직업, NPC 후보 정리용
- `monsters/`: 일반 몬스터, 보스 후보 정리용
- `vfx/`: 스킬 이펙트, 히트 이펙트 후보 정리용
- `items_ui/`: 아이템 아이콘, HUD/UI 후보 정리용
- `maps_tiles/`: 필드/던전 타일맵, 구조물 후보 정리용
- `audio/`: 효과음, BGM 후보 정리용

지금 단계에서는 무작정 다운로드보다, 스타일 통일이 되는 후보를 먼저 추려 두는 데 집중합니다.

검수 포인트:

- 스샷처럼 진짜 게임 느낌이 나는가
- 캐릭터/NPC/몬스터 스타일이 서로 충돌하지 않는가
- 모바일 HUD에 올렸을 때 글자와 아이콘이 살아나는가
- 스킬 이펙트가 빈약하지 않은가
- 맵이 "찍어낸 테스트맵"이 아니라 실제 RPG 맵처럼 보일 수 있는가
